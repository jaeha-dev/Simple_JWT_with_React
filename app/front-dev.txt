+----------------------------------------------------------------------------------------------------------------------+
* Cookie
>> 클라이언트에 보관하는 데이터로 만료 기한을 갖는다.

* Web Storage
>> 웹 스토리지는 HTML5에서 쿠키의 단점을 보완한 기술로 키-값의 형태를 갖는다.
>> 로컬 스토리지와 세션 스토리지는 데이터의 영속성의 차이점을 갖는다.

* Local Storage
>> 웹 브라우저를 종료해도 데이터가 보존되어 다음 번 접속에도 사용할 수 있다.
>> window 전역 객체의 localStorage 컬렉션을 통해 저장과 조회, 삭제가 이루어진다.
   (getItem, setItem, removeItem 등의 메소드 제공)

* Session Storage
>> 웹 브라우저를 종료하면 데이터도 삭제된다. (웹 브라우저 종료 시 세션 스토리지도 삭제됨)
>> window 전역 객체의 sessionStorage 컬렉션을 통해 저장과 조회, 삭제가 이루어진다.

>> 참고: https://www.zerocho.com/category/HTML&DOM/post/5918515b1ed39f00182d3048,
         https://ko.javascript.info/localstorage
+----------------------------------------------------------------------------------------------------------------------+
* props, state
>> 리액트에서 데이터를 다룰 때 사용하는 요소이다.
>> props는 부모 컴포넌트가 자식 컴포넌트에 전달하는 데이터이다.
   (데이터 변경이 필요할 경우 부모 컴포넌트에서 변경되어야 한다.)
>> state는 각 컴포넌트의 상태를 저장하기 위해 사용한다.
   (주의할 점은 생성자에서 state를 사용하고 그외에는 setState를 사용해야 한다.)

>> 참고: https://trustyoo86.github.io/react/2017/11/18/props-state-react.html,
         https://velog.io/@04_miffy/Props-vs-State
